// self-hosted live-set vault

A vault for your DJ live sets, the way Plex is a vault for your movies.

Mixcloud × 1001tracklists × Plex — but for live sets specifically. Lossless storage, time-coded tracklists, provider-enriched metadata, an embeddable player, an installable PWA. Runs on your own hardware, under your own license.

// shipping today · v0.1.2

What works right now on your server.

Ingest

Upload anything, rip from anywhere.

Drag-drop FLAC / WAV / MP3 via resumable tus uploads. Paste a URL and yt-dlp pulls from YouTube, SoundCloud, Mixcloud, Internet Archive, Bandcamp. Drop files into a watch folder and they ingest themselves. Auto-transcoded to Opus, normalized to EBU R128, fingerprinted, dedup-checked.

✓ tus resumable uploads
✓ yt-dlp paste-URL rip
✓ watch-folder auto-ingest
✓ ffmpeg → Opus + EBU R128
✓ chromaprint dedup
Tracklist

Time-coded tracklists, three input modes.

Build per-set tracklists with a time-coded editor. Hit M during playback to add the current track at the current timestamp. Paste a flat tracklist and the parser handles it. Or import from a 1001tracklists URL or an OCR'd image.

[00:00] ID — Opening
[12:32] Surgeon — Bad Hands
[24:48] DJ T-1000 — Inhuman
▸ press M to add at current time
Metadata

Pluggable providers, your keys.

MusicBrainz (free, always on), Discogs (BYO key), AcoustID (BYO key). Per-row Resolve, Bulk resolve, and AcoustID "ID this" for unknown tracks. Your raw data only leaves the box when you enable a provider.

Player

Variable speed. A↔B loop. Lockscreen-ready.

wavesurfer.js with variable speed 0.5×–2× (pitch-preserved), A↔B loop, resume-where-you-left-off per user. MediaSession integration drives the OS lockscreen with artwork + prev/next that jumps tracklist entries.

Engagement

Comments with waveform markers.

Drop comments at any timestamp; they appear as markers across the waveform. @mention crew members and they get an in-app + email notification. Per-set bookmarks. Private notes per user.

3 comments · @mentions land in your inbox
Distribution

Per-user RSS. Embeddable player.

Generate per-user RSS feeds (favorites / recent / everything) with HMAC-signed short-TTL enclosures — paste into any podcast app. Drop the /embed/[slug] URL into any blog post for a public player, toggled per set.

GET /api/feed/favorites/{token}.xml
GET /api/feed/recent/{token}.xml
GET /api/feed/everything/{token}.xml

Installable as a PWA on desktop and mobile. Offline set-detail. Audio cache with an admin-configurable size cap and oldest-first eviction. MediaSession on Android Chrome and iOS Safari shows lockscreen artwork + transport.

Deploy

One image. Three env vars. Port 1970.

v0.1.2 ships as a single all-in-one container running web + worker + watcher under s6-overlay, on port 1970 (year of the first DJ live set). Three required env vars: SECRET_KEY, POSTGRES_PASSWORD, BASE_URL. Multi-arch images on GHCR, SBOM (CycloneDX), cosign keyless signatures.

// quickstart

From curl to "the crew is listening" in 60 seconds.

Three commands. No build. No registration. Default config works on a fresh box.

  1. Pull the example compose.yml and .env from the repo.
  2. Set SECRET_KEY, POSTGRES_PASSWORD, BASE_URL.
  3. docker compose pull && docker compose up -d.
  4. Open BASE_URL (port 1970 by default). Create the admin via the first-run flow. Invite your crew.
setvault@nuc:~ 
# fetch the example compose + env
$ curl -L -o compose.yml \
    https://raw.githubusercontent.com/Bardesss/setvault/main/infra/docker/compose.example.yml
$ curl -L -o .env \
    https://raw.githubusercontent.com/Bardesss/setvault/main/.env.example

# set the three required env vars
$ openssl rand -base64 48
$ $EDITOR .env # SECRET_KEY, POSTGRES_PASSWORD, BASE_URL

# pull + launch
$ docker compose pull
$ docker compose up -d

✓ postgres: ready
✓ redis: ready
✓ tusd: listening on 1080
✓ setvault: http://localhost:1970

# open the URL printed by your reverse proxy
