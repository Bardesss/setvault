Implementation note for Phase 2. Every pattern above is built solely on the variables in tokens.css — change a token and the entire app responds. Light-mode is supported via :root[data-theme="light"] and prefers-color-scheme; reduced motion is honored. The SvelteKit components in Phase 2 should consume these tokens through Svelte's CSS custom-property APIs (no parallel design system); each pattern here maps to one Svelte component.
