01.App screens

Click a card to open the screen full-size. Hover for a hint.

01 · marquee · the most-used screen in the app

Set detail / player view

Sticky audio player with waveform, tracklist sidebar that highlights the currently-playing entry, time-stamped comments as markers, keyboard-driven editing UI, mini-player. The marquee — get this right and the rest follows.

M mark playhead   ·   B bookmark   ·   [ ] step entries   ·   ⇧L loop A↔B
02 · daily-use list

Library / browse

Filter sidebar (genre, year, venue-kind, duration, source), grid + dense-row toggle, applied-filter pills, smart pagination. Filter state lives in the URL — shareable, bookmarkable.
03 · first impression

Home / activity feed

Greeting + stats grid (sets, tracks resolved, IDs needed, storage). Continue listening, recently added, crew activity feed, active jobs panel with live progress bars.
04 · the differentiator

Tracklist editor

Three input modes (live "mark playhead", paste-and-parse messy text, import from URL). Inline editing with provenance tracking, AcoustID candidate list, time-shift dialog, bulk-resolve flow.
05 · mobile + PWA

Mobile player view

Three phone breakpoints — full set view (360 px), library w/ persistent mini-player, OS lockscreen via MediaSession API. Same tokens; collapses cleanly.

How to preview locally:

  1. Open frontend/design/index.html directly in any browser, or…
  2. Run npx serve frontend/design (or any static server) and visit the URL it prints. Live-reload optional.
  3. Each screen is a complete, standalone HTML file. Click any card above to open full-size.

Phase 2 implementation builds against the patterns here. The tokens in assets/tokens.css become the SvelteKit app's design system. Components maps 1:1 to Svelte components. Landing page becomes a SvelteKit static-adapter build deployed to GitHub Pages on push to main.
